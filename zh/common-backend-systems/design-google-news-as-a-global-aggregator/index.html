1. 基础知识与解题思路

本题考察点

Google News 从全球数千家发布商聚合文章，将其聚类成新闻事件，按相关性和时效性进行排名，并为每位用户个性化推荐。难点在于跨语言实时将文章聚类成连贯的新闻事件、分钟级时效性，以及全球规模下的个性化。

其不变量如下：

每篇已发布的文章都在有界时效内被抓取（或推送），去重，与其他关于同一事件的文章聚类，排名，并呈现给相关用户；新闻事件聚类保持连贯性（任何时候一篇文章只属于一个事件）；突发新闻在数分钟内出现。

在 ICT5 / L6+ 级别，面试官考察的是你能否设计实时抓取与摄取、文章聚类（基于 NLP）、每用户排名、多语言处理以及个性化推荐。

60 秒面试框架

口述版本：

我会将 Google News 设计为从发布商→抓取器→文章处理器→聚类引擎→排名→个性化 Feed 的流水线。文章被实时抓取、去重，通过 NLP 相似度聚类成新闻事件，按时效性、来源权威性和用户相关性排名。多语言分别处理。个性化将用户兴趣与编辑策略相结合。有趣的决策在于聚类、实时摄取、排名和个性化。

功能需求

功能性需求：

  • 从发布商聚合文章（RSS、网站地图、爬虫抓取、合作伙伴推送）。
  • 将关于同一新闻事件的文章聚类。
  • 按话题、地区、用户展示热门新闻。
  • 个性化 Feed。
  • 多语言支持。
  • 话题分类（政治、科技、体育等）。

非功能性需求：

  • 文章摄取后→数分钟内可见。
  • 高可用性。
  • 新闻事件聚类稳定，但随着新文章到来而演化。
  • 多地区。

以下内容超出范围，除非被特别询问：

  • 编辑 UI 工具。
  • 订阅 / 付费墙。
  • 新闻通讯分发。

瓶颈：大规模实时聚类 + 个性化推荐。

抓取 / 摄取

多种来源：\n- RSS / 网站地图轮询： 检查发布商 Feed；礼貌限速类似网络爬虫设计。\n- 推送： 合作伙伴发布商发送通知。\n- 发现： 通过网络爬虫信号发现新来源。

目标：顶级发布商实现分钟级时效性；长尾发布商实现小时级时效性。

文章处理

对于每篇新文章：

  1. 提取正文（去除样板内容）。
  2. 检测语言。
  3. 计算向量嵌入（文本→通过 Transformer 生成稠密向量）。
  4. 分类话题（多标签）。
  5. 评估发布商权威性得分。
  6. 交给聚类模块处理。

聚类引擎

实时在线聚类。

解决方案：

  1. 对于每篇新文章，计算向量嵌入。
  2. 与最近新闻事件（最近 N 小时）的质心进行比较。
  3. 若与某个新闻事件质心的相似度大于阈值，则加入该新闻事件。
  4. 否则，以该文章为种子创建新的新闻事件。
  5. 更新新闻事件质心。
  6. 新闻事件在 N 天后老化关闭。

对于多语言聚类：使用跨语言嵌入，或先将内容翻译为标准语言。

边界情况：\n- 文章适合多个新闻事件：选择相似度最高的（或使用子聚类）。\n- 随着文章内容分化，新闻事件随时间分裂：情况罕见；定期重聚类任务处理。\n- 新闻事件合并：发现两个聚类属于同一事件→合并成员文章。

面试用语：

在线聚类：每篇新文章通过嵌入相似度加入现有新闻事件，或开启新事件。质心更新；事件老化。定期重聚类修复漂移。

新闻事件排名器

对每个话题 / 地区 / 用户，对新闻事件进行排名：

信号：\n- 时效性。\n- 来源权威性（高声誉发布商加分）。\n- 事件中文章数量（重要性信号）。\n- 参与度（用户点击）。\n- 用户兴趣匹配度。\n- 编辑信号。

多阶段：廉价预排名→在此基础上的 ML 排名器。

个性化

用户画像包含兴趣、地区、语言、阅读历史。在线排名器融合：\n- 话题偏好（体育 vs 财经）。\n- 来源偏好。\n- 本地相关性。\n- 点击历史。

每日批量更新用户特征。

编辑覆盖

对于突发新闻或内容策划，编辑团队可以：- 将特定报道提升至顶部。- 将报道标记为敏感内容。- 调整信源权威度。

编辑状态优先级高；立即生效可见。

多语言

两种策略：- 按语言独立流水线： 每种语言有各自的聚类和排名流程。用户看到其偏好语言的内容。- 跨语言： embedding 统一；同一事件的英语和西班牙语文章聚合在同一 cluster 中，以用户语言展示。

大多数生产系统结合两者：语言内部聚类，但同一事件的跨语言报道可同时展示两个版本。

扩展与分区

数据 分区策略

报道（Stories） story_id 哈希；按话题、地区建立索引

一致性模型

层级 一致性
文章写入 强一致性
报道成员关系 最终一致性（聚类为异步但快速）
报道排名 最终一致性（定期刷新）
个性化 最终一致
编辑覆盖 强一致性

多区域策略

  • 爬虫按地区部署。
  • 报道存储全局复制。
  • 用户画像存储在归属地区。
  • 排名按地区，结合用户语言偏好。

故障模式与缓解措施

故障模式 缓解措施 监控指标
爬虫无法到达发布方。 重试；回退到备用 feed。 按发布方的拉取失败。
文章被错误聚类。 定期重新聚类；用户反馈。 Cluster 纯度。
耸人听闻内容/错误信息。 信源权威度加权；编辑介入。 被举报的错误信息。
重复报道（聚类分裂）。 定期合并。 重复率。
个性化过滤气泡。 多样性注入。 多样性指标。
报道爆发（突发新闻）。 自动扩展排名；编辑推送。 每报道的文章速率。
跨区域复制。 异步；有界。 延迟。
隐私。 用户退出个性化。 退出合规性。
样板文本提取失败。 多重提取器；回退机制。 提取失败率。
Embedding 服务变慢。 批量处理 + 队列化。 Embedding 队列延迟。

运维指标

正确性：

  • Cluster 纯度（抽样）。
  • 从发布到展示的时延。
  • 错误信息率。

容量：

  • 摄入延迟。
  • 聚类吞吐量。
  • 排名 p99。

业务层面：

  • DAU。
  • 点击率（Click-through rate）。
  • 每用户多样性。

需要指出的权衡

  • 实时聚类以牺牲部分准确性换取新鲜度。
  • 聚类阈值越激进，合并越多，反之拆分越多。
  • 个性化提升参与度，但有过滤气泡风险。
  • 跨语言统一聚类复杂，但用户体验统一。
  • 权威度加权对新发布方不利。
  • 编辑与算法之间的平衡。

常见陷阱与更优做法

陷阱 更优做法
纯批量聚类。 在线增量聚类 + 定期重新聚类。
单语言流水线。 按语言独立流水线 + 跨语言统一。
无信源权威度。 发布方信誉评分。
无个性化。 用户画像 + ML 排名器。
无编辑层。 突发新闻编辑覆盖。
信任客户端点击。 服务端点击验证。

面试用语

Google News 是一条从发布方到爬取、处理、聚类、排名、个性化的流水线。实时聚类通过 embedding 相似度将文章分组为报道；排名器综合新鲜度、权威度、互动度和用户兴趣评分。多语言通过按语言独立流水线 + 跨语言统一 embedding 实现。突发新闻通过编辑覆盖处理。最难的正确性问题是保持 cluster 的连贯性；最难的规模问题是在每日百万文章量下实现实时聚类。

2. 模拟面试

面试官： 设计 Google News。

候选人： 流水线：发布方 → 爬虫/摄入 → 文章处理器（解析、embedding、分类）→ 在线聚类引擎 → 排名 → 个性化 feed。报道是由相关文章组成的 cluster，通过在线 embedding 相似度形成。按语言独立流水线，结合跨语言统一。突发新闻采用编辑覆盖。不变量是：实时新鲜度 + 连贯的报道 cluster + 每用户个性化排名。

面试官： 带我过一遍文章摄入流程。

候选人： 爬虫从发布方抓取新文章（RSS 轮询或推送）。文章处理器提取正文（去除样板文本），检测语言，通过 transformer 计算 embedding，分类话题，评估发布方权威度。聚类引擎将 embedding 与近期报道（最近 N 小时）的质心进行比较；若相似度超过阈值，则加入该报道；否则，创建新报道。更新报道质心。报道排名器重新评分受影响的报道。数分钟内在 feed 中可见。

面试官： 聚类 — 算法？

候选人： 在线聚类。维护近期报道的质心（滑动窗口，通常为最近 1-3 天）。对每篇新文章，计算与所有活跃质心的余弦相似度；若最大值超过阈值，则分配到该报道并更新质心；否则以该文章为种子创建新报道。定期重新聚类任务处理漂移（每隔数小时）：重新评估边界文章，合并意外分裂的 cluster。

面试官： 两个报道应被合并。

候选人： 定期重新聚类任务检测质心非常相似的 cluster。合并：取成员文章的并集，重新计算质心，废弃其中一个 story_id（或选择规范 ID）。编辑可在需要时手动合并或拆分。

面试官： 排名信号？

候选人： 多信号融合。新鲜度（越新越好）。信源权威度（高声誉发布方加权）。报道规模（文章数量多代表重要性，但有边际递减以避免标题党）。互动度（CTR、阅读时长）。用户兴趣匹配。编辑加权。本地相关性。多阶段：廉价预排名剪枝；ML 排名器对候选进行评分。

面试官： 个性化。

候选人： 用户画像：话题兴趣（体育、财经）、偏好信源、地区、语言、阅读历史。在线排名器将全局报道得分与用户亲和度得分融合。多样性注入防止过滤气泡——定期展示用户典型兴趣之外的报道。每日批量更新用户特征。

面试官： 多语言支持。

候选人： 按语言独立流水线聚类（英语文章与英语聚类；西班牙语与西班牙语）。跨语言统一使用多语言 embedding：同一事件的西班牙语文章与英语文章在共享 embedding 空间中得分相似。或先翻译为规范语言再聚类。以用户偏好语言展示；若无该语言文章，可展示其他语言的头条文章。

面试官： 信源权威度。

候选人： 每发布方信誉评分。依据：域名权威度、编辑标准、发布历史、事实准确性信号、用户反馈。高权威度信源在排名中排名更高。对新发布方有影响——需要时间积累权威度。编辑可在突发情况下策划权威度。

面试官： 错误信息。

候选人： 信源权威度是主要防护措施。低权威度信源降权。特定错误信息模式（深度伪造、已知虚假声明）由 ML 分类器标记。新兴案例由编辑介入处理。叠加事实核查标签。用户可举报；举报结果反馈至编辑审核。

面试官： 突发新闻。

候选人： 实时流水线确保快速展示——文章在数分钟内摄入。多家发布方报道时，突发事件的新报道迅速形成。编辑可将报道推送至顶部。排名器对高发布速率的报道加权。推送通知可向用户推送头条报道。

面试官： 多地区。

候选人： 爬虫按地区部署；合作方推送全球接受。报道存储全局复制，支持跨地区浏览。用户画像存储在归属地区。排名按地区（地区兴趣加权），结合用户语言偏好。

面试官： 隐私/个性化。

候选人： 用户退出个性化后获得非个性化的算法 feed。阅读历史保留期限有界。用于全局趋势的信号经匿名化处理。遵从每用户退出设置。

面试官： 总结。

候选人： 流水线：爬取 → 处理 → 聚类 → 排名 → 个性化。在线聚类通过 embedding 相似度将文章分组为报道。多阶段排名，融合多维信号（新鲜度、权威度、互动度、个性化）。按语言聚类 + 跨语言统一，支持多语言覆盖。突发新闻编辑覆盖。信源权威度防御错误信息。多地区，全局报道复制。最难的正确性问题是大规模下的 cluster 一致性；定期重新聚类处理漂移。

3. 练习备注

五分钟版本

  1. 明确范围：聚合、聚类、新鲜度、个性化。
  2. 不变量：实时新鲜度，连贯的 cluster。
  3. 流水线：爬取 → 处理 → 聚类 → 排名 → 个性化。
  4. 通过 embedding 相似度在线聚类。
  5. 多阶段排名，融合权威度与个性化。
  6. 按语言独立流水线 + 跨语言统一。
  7. 编辑覆盖；错误信息防御。
  8. 多地区，全局报道复制。

一分钟结尾陈述

Google News 是发布方 → 爬取 → 处理 → 聚类 → 排名 → 个性化。在线聚类通过 embedding 相似度将文章分组为报道。多阶段排名信号：新鲜度、信源权威度、互动度、用户个性化。多语言通过按语言独立流水线 + 跨语言统一 embedding 实现。突发新闻编辑覆盖。信源权威度防御错误信息。最难的正确性问题是 cluster 一致性；最难的规模问题是在每日百万文章量下实现实时聚类。

最佳面试用语

  • "在线聚类：每篇文章加入最相似的报道，或作为种子创建新报道。"
  • "多阶段排名；权威度加权是关键信号。"
  • "个性化将用户兴趣与全局排名融合；多样性注入防止过滤气泡。"
  • "多语言通过按语言独立流水线 + 跨语言统一 embedding 实现。"
  • "突发新闻编辑覆盖；信源权威度防御错误信息。"
