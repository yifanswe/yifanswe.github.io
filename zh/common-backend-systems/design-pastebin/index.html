1. 基础知识与解题思路

本题考察点

Pastebin 允许用户粘贴一段文字并获取一个短链接。其结构与 TinyURL 类似，但有几个重要区别：payload 更大（KB 到 MB 级），过期功能很常见，代码语法高亮/格式化是预期功能，隐私和滥用问题也是真实存在的。

其不变量如下：

每个成功发布的 paste 在过期前都应被持久存储并可通过其唯一 URL 访问；该 URL 应具有足够的不可猜测性，为未列出的 paste 提供合理的隐私保障；读取速度要快；滥用行为（恶意软件、凭据泄露）应在检测到时被过滤或删除。

在 ICT5 / L6+ 级别，面试官考察的是你能否设计一个"一写多读"的文本存储系统，处理过期逻辑，将元数据与内容存储分离，管理滥用行为，并分析 paste 内容热门时的读取热点问题。

60 秒面试框架

口述版本：

我会将 Pastebin 设计为以生成的 paste_id 为键的"一写多读"内容存储系统，元数据存放在高速 KV 存储中，内容存放在对象存储中。由于 paste 内容不可变，读取请求由 CDN 前置处理。关键决策包括：ID 生成、过期处理、隐私级别（公开/未列出/私密）、滥用检测，以及热门 paste 的处理。

首先明确需求

功能性需求：

  • 用户粘贴文字，可选择指定语法语言提示。
  • 获取一个短小的唯一 URL。
  • 访问 URL 查看 paste，带语法高亮。
  • 可选过期时间（10 分钟、1 天、1 个月、永久）。
  • 隐私级别：公开（已索引）、未列出（仅限 URL 访问）、私密（需要登录）。
  • 作者可编辑（可选）或删除。
  • 评论功能（有时提供）。

非功能性需求：

  • 亚秒级粘贴读取。
  • 持久存储；无数据丢失。
  • 支持随时间增长至数十亿条粘贴。
  • 处理热点粘贴读放大问题。
  • 滥用处理：恶意软件、凭证泄露。
  • 多地区。

以下内容超出范围，除非被特别询问：

  • 协同编辑。
  • 基于账户的文件夹。
  • 高级访问控制列表（ACL）。

规模假设

领域 假设
粘贴数/天 数百万。
读取数/天 写入量的100倍。
平均粘贴大小 5-50 KB；部分达 MB 级别。
活跃粘贴数 随时间累计达数十亿。
Latency（延迟） 读取 < 200 ms；写入 < 500 ms。
热点粘贴 可获得数百万次浏览（例如在 Twitter 上分享）。
留存期 永久有效（可选设置过期时间）。

瓶颈在于热点粘贴读取和总存储量的持续增长。

核心产品模型

界面 体验 需求
创建 粘贴内容，选择选项，获取 URL。 亚秒级。
Read 访问 URL，查看粘贴内容。 亚秒级；可缓存。
管理 作者编辑/删除。 范围有限。
滥用 举报；审核。 人工 + 自动化。

核心组件

组件 职责 关键细节
API 网关 鉴权，限速。 标准。
创建服务 验证，生成 id，写入。 幂等。
Id 生成器 唯一短 paste_id。 范围计数器或随机方式。
Metadata store paste_id -> {content_ref, expiry, owner, privacy, syntax} 快速 KV。
内容存储 实际粘贴内容。 对象存储。
CDN 缓存已渲染/原始粘贴。 激进 TTL，因为粘贴是不可变的。
读服务 获取元数据；提供内容；检查是否过期。 无状态。
滥用处理流水线 扫描恶意软件、凭证、受版权保护的内容。 异步处理。
过期清理器 删除已过期的粘贴。 后台运行。
搜索（可选） 公开粘贴索引。 最终一致性。

数据模型

粘贴元数据

PasteMeta(
  paste_id,
  content_ref,           // object storage URI
  content_hash,
  size_bytes,
  syntax_lang,
  privacy,               // public, unlisted, private
  owner_id?,
  created_at,
  expires_at?,
  views_count_approx,
  status,                // active, deleted, blocked
  abuse_status,
  version
)

粘贴内容

存储在对象存储中；通过以下方式寻址： content_ref.

滥用决策

AbuseDecision(
  paste_id,
  decided_at,
  reason,                // malware, credentials, copyright, spam
  source,                // scanner, manual_review, partner
  status                 // benign, suspicious, blocked
)

API

POST   /pastes
       Body: { content, syntax, privacy, expires_at? }
       -> { paste_id, url, expires_at }
GET    /pastes/{paste_id}
       -> { content, syntax, created_at, ... }
GET    /pastes/{paste_id}/raw
       -> text/plain
DELETE /pastes/{paste_id}  (owner only)
POST   /pastes/{paste_id}/report

Id 生成

与 TinyURL 相同：范围计数器预占或随机 + 条件插入。

  • 使用 Base62 编码生成紧凑 URL。
  • 通常为 7-8 个字符（可寻址空间为 3.5T 至 218T）。
  • 匿名隐私场景优先选用随机方式（不可猜测）。
  • 对于公开粘贴，顺序 id 可接受，但会暴露数据量。

写入路径

  1. 客户端 POST 粘贴内容及选项。
  2. API 网关进行鉴权（已登录时）并对匿名用户限速。
  3. 创建服务生成 paste_id。
  4. 计算内容哈希。
  5. 将内容上传至对象存储。
  6. 插入 PasteMeta，其中包含 content_ref 指向对象。
  7. 将滥用扫描任务入队。
  8. 返回 paste_id 和 URL。

对于大型粘贴，使用签名 URL 直接向对象存储发起分片上传，然后提交元数据。

读取路径

  1. 客户端访问 /{paste_id}.
  2. CDN 检查缓存；命中时直接返回已渲染/原始粘贴。
  3. 未命中时，请求到达读取服务。
  4. 读取服务：\n - 根据 paste_id 查找 PasteMeta。\n - 检查是否过期及滥用状态。\n - 从对象存储获取内容。\n - 按需渲染（语法高亮）或返回原始内容。
  5. CDN 按不超过过期时间的 TTL 进行缓存。

对于不可变粘贴，CDN TTL 可以非常长（数小时至数天）。

缓存策略

粘贴是不可变的；缓存策略简单直接。

  • Edge CDN 缓存粘贴内容，设置 Cache-Control: max-age=N.
  • TTL 以过期时间为上限： min(default_ttl, expires_at - now).
  • 以下事件触发缓存失效：
  • 作者删除。
  • 滥用下架。
  • 编辑（如支持）。

多层架构：CDN → 区域读取服务 → 元数据存储 + 内容存储。

过期时间

  • expires_at 在创建时设置。
  • 读取路径检查过期；已过期的粘贴返回 410 Gone。
  • 后台清理器定期按时间桶删除过期的元数据和内容。
  • 最终一致性：已过期的粘贴可能在清理运行前短暂仍可访问。

隐私

隐私 行为
公开 可被搜索索引；出现在列表中。
不公开列出 仅通过直接 URL 访问；不被索引。
私有 需要鉴权；仅限所有者访问。

对于公开粘贴，搜索索引定期摄取公开粘贴。

滥用处理流水线

Pastebin 常被用于传播恶意软件、凭证泄露、代码外泄。防御措施：

  • 上传后运行内容扫描器：已知恶意签名、凭证正则（密码、令牌）、URL 黑名单比对。
  • 信誉机制：低信任用户（匿名、新账户）触发激进扫描。
  • 用户举报触发人工审核。
  • 高置信度检测时自动封锁：状态 blocked；读取返回拦截页面。
  • 遵守下架通知（DMCA 等）。
  • 封锁时缓存失效。

扩展与分区

数据 分区键 原因
元数据 paste_id 哈希值 直接查找。
内容 paste_id 哈希；对象存储 每个粘贴的 blob。
过期索引 按时间桶分区 expires_at 清理器扫描。
用户粘贴列表 owner_id 哈希值 "我的粘贴"列表。
公开索引 哈希分区 搜索。

一致性模型

层级 一致性 原因
元数据写入 强一致性 真实数据来源。
内容不可变 首次写入强一致性 无编辑（或新建版本）。
CDN 缓存 最终一致 对不可变内容可接受；下架时显式失效。
滥用判定 最终一致 异步扫描。
过期时间 最终一致 清理延迟可容忍。

多区域策略

  • 元数据全局复制。
  • 内容存储于主区域；复制到其他区域。
  • CDN 边缘节点遍布全球。
  • 区域本地创建，全局复制。

故障模式与缓解措施

故障模式 缓解措施 监控指标
热点读取（病毒式传播粘贴）。 CDN 吸收；多层架构。 源站出口流量；CDN 命中率。
存储增长。 过期清理器；冷数据分层。 按租户存储。
滥用内容。 扫描器 + 人工审核；下架处理。 封禁生效时间。
下架后缓存过时。 失效事件；下架前 TTL 短化。 过时服务计数。
Id 碰撞（随机）。 条件插入 + 重试。 碰撞率。
作者删除与查看操作竞争。 软删除 + 缓存失效。 删除与查看的顺序问题。
大型粘贴上传失败。 可续传的分片上传。 上传失败率。
垃圾粘贴。 限制匿名速率；CAPTCHA 验证。 匿名创建速率。
搜索索引延迟。 异步；有界。 索引延迟。
跨区域复制。 异步；有界。 复制延迟。

运维指标

正确性：

  • 未过期粘贴的读取成功率（~100%）。
  • 下架后的过时服务。
  • 存储增长。
  • 滥用从发现到封锁的时间。

容量：

  • 创建 QPS。
  • 读取 QPS，CDN 命中率。
  • 存储总量。

业务层面：

  • 活跃粘贴数；每条粘贴的浏览量。
  • 滥用率。

需要指出的权衡

  • 随机 id vs 顺序 id：不可猜测性 vs 成本可预测性。
  • 激进 CDN TTL vs 下架速度；显式失效作为桥接。
  • 长期保留 vs 存储成本。
  • 激进滥用过滤 vs 误判率。
  • 严格限速 vs 合法用户体验。
  • 更强隐私模式 vs 用户体验（登录要求）。

常见陷阱与更优做法

陷阱 更优做法
将内容存储在元数据 DB 中。 独立的内容存储（对象存储）。
同步滥用扫描。 异步 + 占位拦截页面；事后下架。
为不公开列出的隐私使用顺序 id。 使用随机 id 保证不可猜测性。
忽略过期处理。 后台清理器 + 读取时检查。
单层 CDN。 多层 + 缓存失效。
无滥用处理。 扫描器 + 举报 + 人工审核流水线。

面试官可能深入探讨的方向

深入方向 优秀回答要点
Id 生成 范围计数器 vs 随机；碰撞重试。
热点读取 CDN 多层；预热。
滥用 异步扫描；缓存失效；下架流水线。
过期时间 清理器 + 读取时检查。
隐私级别 公开/不公开列出/私有；访问权限检查。

Staff 级别回答检查清单

  • 元数据与内容分离存储。
  • CDN 用于不可变内容。
  • 异步滥用扫描，含下架缓存失效。
  • 过期：读取时检查 + 后台清理。
  • 隐私级别。
  • 多区域复制。
  • 匿名创建限速。

面试用语

Pastebin 是一个一次写入的内容存储系统，元数据存于快速 KV，内容存于对象存储，读取通过 CDN 提供。粘贴是不可变的，因此缓存简单；运营挑战在于滥用处理和粘贴病毒式传播时的热点读放大。

2. 模拟面试

面试官： 设计 Pastebin。

候选人： 三个核心部分：元数据 KV，将 paste_id 映射到 {content_ref, expiry, privacy, owner}；内容对象存储，保存实际文本；CDN 负责读取服务。创建时同时写入两者；读取时优先从 CDN 缓存获取，缺失时回源。滥用扫描在创建后异步运行；下架时失效缓存。不变量是持久存储和在过期前可检索，同时及时封锁滥用内容。

面试官： Id 生成方案？

候选人： 两种选择。范围计数器预占（类似 TinyURL）：每个创建节点预先认领一段整数范围，编码为 base62。顺序可预测，碰撞风险低。或随机 7-8 位 base62 字符 + 条件插入。对于 Pastebin，随机方式更优，因为不公开列出的隐私依赖于 URL 不可猜测。顺序 id 会让任何人都能枚举。

面试官： 一条粘贴在 Twitter 上爆火——每秒 10 万次读取。

候选人： CDN 几乎完全吸收所有流量。该粘贴是不可变的；缓存 TTL 可达数小时。Edge POP 缓存渲染后或原始内容。源站只承受极少量请求。我们监控每条粘贴的读取速率，必要时预热 POP。若源站被击穿，也只是单次元数据 + 内容读取。

面试官： 滥用——凭证泄露。

候选人： 异步内容扫描器。使用正则匹配凭证、API 密钥、已知泄露凭证模式。使用 ML 分类器识别恶意软件风格的混淆文本。高置信度检测时，设置 abuse_status=blocked。发出 CDN 失效指令。读取服务返回拦截页面而非内容。边界情况进行人工审核。用户举报的粘贴进入审核队列。我们衡量从发现到封锁的时间。

面试官： 过期机制？

候选人： expires_at 在创建时设置。读取服务在每次读取时检查过期——若已过期，返回 410 Gone 且不获取内容。后台清理器定期按时间桶扫描，删除过期的元数据和内容。读取时检查防止清理器延迟带来的问题；清理器负责回收存储空间。

面试官： 隐私模式？

候选人： 公开：可被搜索索引，显示在首页列表。不公开列出：仅通过 URL 访问，不被索引。私有：需要鉴权，仅限所有者。隐私标志存于 PasteMeta；读取服务负责强制执行。随机 id 使不公开列出的粘贴具有合理的隐私性，因为 URL 不可猜测。

面试官： 那编辑功能呢？

候选人： 两种选择。要么粘贴是一次写入不可变（编辑意味着创建新粘贴并附带跳转链接），要么每条粘贴保留版本历史。Pastebin 传统上是不可变的；若支持编辑，则存储版本链，读取时返回最新版本。版本管理会使缓存略微复杂。

面试官： 存储增长问题？

候选人： 随着时间推移产生数十亿条粘贴内容。将旧的、很少访问的粘贴分级存储到冷对象存储中。过期清扫器删除用户选择过期的内容。软删除作者删除的内容（短暂保留用于审核）。定期对孤立内容 blob 进行垃圾回收。

面试官： 多区域？

候选人： 元数据在全球范围内复制，以便快速读取粘贴内容的存在状态和访问检查。内容存储在主区域，并对高流量粘贴进行跨区域复制。全面使用 CDN。创建操作落在最近的区域，并进行异步复制。

面试官： 如何防止有人刷一百万条粘贴内容？

候选人： 在网关层设置速率限制：匿名用户受严格限制，已登录用户相对宽松。匿名突发请求需通过 CAPTCHA 验证。按 IP 和账户设置配额。对内容进行已知垃圾签名的模式检测。账户信誉反馈到速率限制中。

面试官： 总结。

候选人： Pastebin 是一个一写多读的内容存储系统。元数据存储在快速 KV 中，内容存储在对象存储中，读取通过 CDN 加速。随机 ID 使未公开链接难以猜测。异步滥用扫描，下架时使缓存失效。通过读取时检查和后台扫描实现过期处理。隐私级别在读取时强制执行访问控制。多区域部署，全局元数据与区域主要内容相结合。最困难的运维问题是滥用和病毒式热读，两者均通过异步扫描和激进的 CDN 缓存来应对。

3. 练习备注

五分钟版本

  1. 澄清：文本载荷、过期时间、隐私设置、滥用防护。
  2. 不变量：在过期前保持持久性和可检索性；阻止滥用。
  3. 元数据 KV + 内容对象存储。
  4. 使用随机 id 保证不可猜测性。
  5. CDN 激进缓存（不可变内容）。
  6. 异步滥用扫描 + 下架时使缓存失效。
  7. 过期：读取时检查 + 清扫器。
  8. 隐私级别在读取时强制执行。

一分钟结尾陈述

Pastebin 是一个一写多读的内容存储系统，元数据存储在快速 KV 中，内容存储在对象存储中，前端通过 CDN 加速。随机 ID 为未公开内容提供合理的不可猜测性。滥用内容通过异步过滤处理，下架时使缓存失效。过期时间在读取时检查，并通过后台清扫处理。病毒式热读由 CDN 吸收。多区域复制保持热路径在各地快速响应。

  • "使用随机 ID 提高不可猜测性；基于计数器的 ID 会泄露数据量。"
  • "异步滥用扫描，下架时使缓存失效。"
  • "过期是读取时检查加上后台清扫。"
  • "元数据与内容分离，使我们能够独立扩展存储。"
