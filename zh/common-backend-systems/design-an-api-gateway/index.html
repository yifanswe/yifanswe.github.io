1. 基础知识与解题思路

本题考察点

API 网关是服务网格前的单一入口点。每个公共请求在到达任何后端之前都要经过它。难点在于要足够快（亚毫秒级开销）、安全（认证、防滥用、限流），且支持动态配置（路由变更无需重启）。

其不变量如下：

每个进入系统的请求都经过认证、授权、限流，并以可预测的延迟开销路由至正确的后端，同时网关绝不能成为拖垮整个平台的单点故障。

在 ICT5 / L6+ 级别，面试官考察的是你能否将数据平面（逐请求处理）与控制平面（配置、密钥、策略）分离，处理不中断连接的配置变更，集成可观测性和限流子系统，并判断哪些功能应放在网关中，哪些应放在专用服务中。

60 秒面试框架

口述版本：

我会将 API 网关设计为水平可扩展的数据平面加一个独立的控制平面。数据平面负责 TLS 终止、请求认证、限流执行、路由至后端以及输出可观测性数据。控制平面负责路由定义、认证策略和密钥，并将版本化配置推送至数据平面节点。关键决策在于：哪些功能放在网关、配置更新如何安全传播，以及如何将 p99 开销控制在最低。

首先明确需求

功能性需求：

  • TLS 终止；协议支持（HTTP/1.1、HTTP/2、gRPC、WebSocket）。
  • 认证：API 密钥、OAuth、JWT、mTLS。
  • 授权：scope/role 检查；每条路由的策略。
  • 按身份和路由执行限流和配额。
  • 请求/响应转换：Header 重写、Body 重映射、Schema 校验。
  • 路由：基于路径、基于 Header、基于权重（用于金丝雀/A-B 测试）。
  • 服务发现集成；健康检查。
  • 适当情况下缓存响应。
  • 每条路由的日志记录和链路追踪上下文传播。

非功能性需求：

  • 单位数毫秒的 p99 开销。
  • 每个网关集群 100 万+ QPS。
  • 零停机配置更新。
  • 数据平面节点单点故障不对用户产生可见影响。
  • 多租户：不同团队管理各自的路由和策略。

以下内容超出范围，除非被特别询问：

  • L4 负载均衡（假设前方已有 L4 LB）。
  • WAF 规则编写（集成，不自建）。
  • 服务网格 Sidecar（相关但不同的问题）。

规模假设

领域 假设
流量 每集群峰值 100 万请求/秒。
路由 1 万至 10 万条不同路由。
租户 数百个团队。
配置变更 每天数百次变更。
延迟开销 网关增加的延迟 < 5 ms p99。
可用性 每地区 99.99% 可用性。
多区域 每地区部署网关集群。

瓶颈在于每次请求的 CPU 处理（TLS 握手摊销、JWT 解析、路由查找）以及配置变更的控制平面扇出。

核心产品模型

层级 职责 需求
数据平面 每请求：TLS 终止、认证、限流、路由、转换、观测。 无状态；水平可扩展；每个过滤器开销低于毫秒级。
控制平面 管理路由、策略、密钥；将配置推送至数据平面。 强一致性；版本化；可回滚。
可观测性 每个请求输出日志、指标、链路追踪。 异步；绝不阻塞数据路径。
开发者体验 自助式路由注册、测试、部署。 版本化配置、预演（dry-run）、分阶段发布。

核心组件

组件 职责 关键细节
L4 负载均衡器 将连接分散至数据平面节点；用 Anycast 进行地区导流。 DSR、ECMP 或云 LB。
数据平面节点（Envoy/NGINX 风格） 通过过滤链进行逐请求处理。 无状态；主要是 CPU 密集型；通过异步 I/O 实现高并发。
过滤链 有序过滤器列表：TLS、认证、限流、路由、转换。 可插拔；支持每条路由配置。
控制平面 API 路由、策略、密钥的增删改查。 版本化、可审计、角色受限。
配置分发 将配置快照推送至数据平面节点。 xDS 风格的流式传输，或带版本控制的轮询。
密钥存储 保存私钥、JWT 验证密钥、API 密钥哈希。 定期轮换；绝不记录日志。
限流子系统 每身份每路由的计数器。 参见专项限流器设计。
服务发现 将逻辑服务映射到后端端点。 健康检查；加权。
可观测性 Sidecar 流式传输日志、指标、链路追踪。 解耦；绝不阻塞请求路径。
WAF 集成 可选，在过滤链早期运行。 可内联或带外部署。

请求路径（数据平面）

处理每个传入请求：

  1. 接受连接。 L4 LB 已将 TCP 导向某个数据平面节点。
  2. TLS 终止。 使用 SNI 选择正确的证书；使用会话复用摊销握手成本。
  3. 协议解析。 HTTP/1.1、HTTP/2 或 gRPC。
  4. 路由匹配。 路由模式的编译前缀树或基数树；O(log n) 或更优。
  5. 认证过滤器。 验证 JWT 签名（对照缓存的 JWKS）、查找 API 密钥哈希，或验证 mTLS 证书链。失败则拒绝。
  6. 授权过滤器。 根据路由策略校验作用域/角色，验证失败则拒绝请求。
  7. 限流过滤器。 递增计数器；超出配额则拒绝。（详见专项限流器设计。）
  8. 转换过滤器。 添加/删除请求头、重写路径、校验请求 schema。
  9. 缓存过滤器。 若响应可缓存且已存在，则直接返回。
  10. 后端选择。 从服务发现中选取后端实例；支持加权或最少连接策略。
  11. 转发请求。 复用或新建上游连接并发送请求。
  12. 响应处理。 应用响应转换；视情况缓存；输出日志/指标/链路追踪。
  13. 返回客户端。

每个过滤器的延迟预算均在亚毫秒级，整条链路应将 p99 控制在 5 ms 以内。

配置更新路径（控制平面）

当开发者更新一条路由时：

  1. 控制平面 API 接收变更，并对 schema 和策略进行校验。
  2. 写入版本化配置存储（例如 etcd 或带快照功能的数据库）。
  3. 生成一个新的 ConfigSnapshot 包含完整状态。
  4. 通过流式 RPC（xDS 风格）推送至所有数据平面节点。
  5. 每个节点以原子方式替换编译后的配置；正在处理中的请求继续使用旧配置，新请求使用新配置。
  6. 节点以已应用的版本号进行 ack；控制平面跟踪全舰队的覆盖情况。

这是来自 Envoy/Istio 的标准控制平面/数据平面模式。核心保证：配置更新永远不会中断正在进行的连接。

对于高风险变更，请使用 分阶段发布：

  • 首先推送至 1% 的数据平面节点。
  • 监控错误率。
  • 若健康，继续推送至 10%、50%、100%。
  • 错误激增时自动回滚。

面试用语：

配置分发是网关中最难保证正确性的问题。数据平面在安全策略变更等关键更新期间绝不能使用过期配置，但同时在配置切换时也不能中断正在处理中的请求。

认证方式

方式 工作方式 权衡
API key 预共享密钥；网关对其进行哈希后在缓存中查找。 简单易用；密钥轮换属于运维操作。
JWT 客户端提交签名令牌；网关使用公钥验证签名。 无状态验证；吊销需要额外检查。
OAuth2 token introspection 网关对每个请求调用认证服务。 权威性强，但会增加延迟；建议对短期结果进行缓存。
mTLS 在连接层进行双向 TLS 认证；身份来自证书。 身份验证强；客户端配置复杂。

缓存策略：

  • JWKS 公钥按 issuer 缓存，设置 TTL。
  • OAuth introspection 结果以短 TTL 缓存。
  • API key 哈希表通过控制平面推送刷新。

限流

将限流委托给专用的限流服务（详见单独设计），而非在网关内部实现：

  • 网关通过快速 RPC（或本地 sidecar）对每个请求调用限流器。
  • 网关在本地缓存近期决策，以减少热点 key 的调用次数。
  • 降级模式：若限流器不可达，则按路由策略选择 fail open 或 fail closed。

可观测性

每个请求产生：

  • 访问日志： 低流量路由默认 100% 采样；高流量路由采用自适应采样率。
  • 指标： 请求数、延迟直方图、状态码、路由、身份、区域。
  • Trace span： 传播 trace 上下文（traceparent）；为网关跳创建一个 span。

以上数据异步发送至 sidecar 或本地 agent，请求路径不会因可观测性写入而阻塞。

扩展与分区

层级 扩展性
数据平面 水平扩展；无状态；L4 LB 对连接进行负载均衡。
控制平面 有状态（配置存储）；通过共识协议进行副本复制。数据平面读多写少。
限流存储 按 key 分片；独立服务。
密钥存储 认证后以只读副本形式同步至数据平面。

一致性模型

关注点 一致性 原因
路由配置 在数据平面间最终一致；受配置推送延迟约束 适用于非紧急变更。
紧急策略变更（例如封禁某个 key） 有界陈旧目标：全球范围内 10 秒以内 安全敏感；需监控传播情况。
认证密钥 在轮换窗口内最终一致 旧密钥在重叠期内保持有效。
限流计数器 近似计数（详见限流器设计） 可接受。

故障模式与缓解措施

故障模式 缓解措施 监控指标
数据平面节点在请求处理过程中崩溃。 L4 LB 将节点移除；客户端重试；幂等操作是安全的。 每节点错误率。
控制平面宕机。 数据平面无限期运行于最后已知配置。 配置版本偏差；控制平面可用性。
错误配置发布。 分阶段发布；错误激增时自动回滚；通过版本化配置手动回退。 每版本错误率。
后端宕机。 健康检查将其从池中移除；熔断器停止向其发送流量；若已配置则故障转移至备用后端。 后端健康状况。
TLS 证书过期。 自动续期流水线；在过期前充裕时间发出告警。 每路由的证书过期时间。
限流器宕机。 按路由配置降级模式；回退至本地计数器。 限流器可用性。
认证服务宕机。 缓存的 JWT 验证继续生效；introspection 失败则按路由配置选择 fail open 或 fail closed。 认证子系统健康状况。
DDoS 攻击网关。 L4 LB 或 CDN 进行吸收；网关设有连接数限制；WAF 拦截已知攻击特征。 连接数、TLS 握手速率。
后端响应缓慢导致级联问题。 按路由配置超时；熔断器；舱壁隔离。 后端 p99；熔断器开启次数。
单个慢速过滤器导致的尾延迟。 按过滤器监控延迟；过滤器超时设置。 过滤器延迟直方图。

运维指标

正确性：

  • 数据平面间配置版本偏差。
  • 从策略变更到全舰队完成传播的时间。
  • 拒绝原因分布（认证失败、限流、4xx、5xx）。

容量：

  • 每节点 QPS；CPU 使用率。
  • 连接数；每秒新建连接数。
  • TLS 握手速率。
  • 后端连接池利用率。

每路由指标：

  • 每路由延迟 p50/p99。
  • 每路由错误率。
  • 每路由缓存命中率。

多区域策略

  • 每个区域运行独立的网关集群。
  • L4 anycast 或 geo-DNS 将用户路由至最近的区域。
  • 路由和策略是全局配置，并被复制至每个区域的控制平面。
  • 通过 DNS 或撤回 anycast 广播实现按区域故障转移。
  • 对于 active-active 后端，网关仅在本地后端不健康时才考虑跨区域路由。

需要指出的权衡

  • 过滤器链深度会增加延迟；应最小化默认过滤器，并按路由进行按需启用。
  • 集中式限流精度高但有往返开销；本地降级更快但精度较低。
  • 激进的配置推送可加快传播速度，但存在关联性发布失败的风险。
  • 在网关做 TLS 终止可集中管理证书，但会将 CPU 负载集中到网关节点。
  • 缓存响应有助于性能提升，但跨节点的缓存失效较难处理。
  • 在网关中集成认证可简化后端，但会将认证变更与网关部署耦合在一起。

常见陷阱与更优做法

陷阱 更优做法
单一的整体配置文件。 带分阶段发布的版本化快照。
请求路径上的同步日志记录。 异步写入 sidecar/本地 agent。
有状态的网关节点（会话）。 无状态数据平面；状态存储于专用服务。
每请求认证 introspection。 缓存 JWT 验证；仅在需要时进行 introspection。
所有路由共用同一条过滤器链。 每路由过滤器列表；延迟编译。
单一控制平面节点。 带共识的副本化控制平面。

面试官可能深入探讨的方向

深入方向 优秀回答要点
配置分发 xDS 风格流式推送；版本化快照；原子替换；分阶段发布。
认证流水线 JWT + JWKS 缓存；mTLS；OAuth introspection 缓存。
限流器集成 RPC 调用专用服务；本地降级；按路由配置降级模式。
可观测性 异步指标/链路追踪；采样；trace 上下文传播。
热点路由处理 连接池化、按路由熔断器、自动扩缩容信号。
多区域 Geo-DNS 或 anycast；按区域集群；副本化控制平面。

Staff 级别回答检查清单

  • 将数据平面与控制平面分离。
  • 带原子替换的版本化配置快照。
  • 无状态数据平面；水平扩展。
  • 认证、限流、可观测性作为过滤器，支持按路由配置。
  • 高风险变更采用分阶段发布。
  • 按路由配置超时、重试、熔断器。
  • 异步可观测性。
  • 多区域部署为独立集群，具备副本化控制平面。

面试用语

API 网关是经典的数据平面/控制平面分离架构。数据平面无状态，通过避免对可能故障的组件进行同步调用来保持高速；控制平面拥有真实状态并传播版本化快照。每个过滤器都有延迟预算，网关永远不是瓶颈所在。

2. 模拟面试

面试官： 设计一个 API 网关。

候选人： 我将设计两个部分：一个处理每个请求工作（TLS、认证、限流、路由、转换）的无状态数据平面，以及一个拥有路由、策略和密钥并将版本化配置推送至数据平面的控制平面。不变量是过滤器开销亚毫秒级，且永远不是单点故障。

面试官： 配置传播是如何工作的？

候选人： 控制平面 API 写入版本化存储。它生成一个 ConfigSnapshot 包含完整状态，对其签名，并通过流式 RPC（xDS 风格）推送至每个数据平面节点。每个节点以原子方式替换编译后的配置；正在处理中的请求继续使用旧配置，新请求使用新配置。节点以已应用的版本号进行 ack，控制平面跟踪全舰队的覆盖情况。

面试官： 一名开发者推送了一条错误路由，如何避免平台崩溃？

候选人： 分阶段发布。变更首先部署到 1% 的数据平面节点；我们监控错误率几分钟。若健康，则扩展至 10%、50%、100%。若检测到错误激增，则自动回滚至前一版本。开发者也可通过版本化配置存储手动回退；每个快照都有一个 id。

面试官： 如何以最快速度传播紧急安全变更，例如"立即封禁此 API key"？

候选人： 目标在 10 秒内完成全球传播。控制平面通过流式 RPC 立即推送；每个数据平面节点收到后即刻应用。我们明确监控一个 config_propagation_latency 指标。对于封禁已知恶意凭证，我们维护一个拒绝列表，该列表与路由配置分开传播，并在每次认证检查时被查阅。

面试官： 认证是如何工作的？

候选人： 支持多种模式，按路由配置。JWT 最为常见：客户端发送令牌，网关使用缓存的 JWKS 公钥验证签名。JWKS 缓存定期刷新。对于 API key，网关进行哈希后在由控制平面支撑的缓存中查找。对于 mTLS，L4 LB 或网关负责终止并提取客户端证书。OAuth introspection 开销最大——我们调用认证服务并以短 TTL 缓存结果。

面试官： 限流逻辑放在哪里？

候选人： 我将其委托给专用的限流服务，而非在网关中自行实现。网关对每个请求向限流器发起 RPC 调用。对于热点 key，网关有本地近似机制：一个进程内小型令牌桶，用于吸收负载并异步与中央限流器进行对账。若限流器不可达，则由路由的 fail-mode（open 或 closed）决定处理方式。

面试官： 在这个 QPS 下 TLS 怎么处理？

候选人： TLS 握手是网关最昂贵的操作。我们尽可能使用会话复用和 TLS 1.3 零往返复用。在可用时利用硬件加速。通过 SNI 按路由选择证书。证书轮换是自动化的；密钥存储推送新证书，网关无需重启即可加载。

面试官： 后端服务响应缓慢时，如何控制影响范围？

候选人： 按路由超时：若后端在 N ms 内未响应，网关返回 504 并释放连接。熔断器：在超过失败或慢响应阈值后，网关停止向不健康的后端实例发送流量。舱壁隔离：每个后端的连接池有上限，因此单个慢速后端不会耗尽网关的全部并发能力。

面试官： 如何处理 WebSocket 和长连接？

候选人： 使用不同的过滤器链。认证过滤器在连接建立时运行；限流可在初始升级时运行，也可能对每条消息运行。网关与后端保持持久化的上游连接；客户端断开时会向后端传播。连接排空：在部署期间，网关停止接受新的 WebSocket 连接，并等待现有连接完成或迁移。

面试官： 如何应对每天十亿次请求的可观测性需求？

候选人： 异步可观测性：每个请求将结构化事件发送至 sidecar 或本地 agent，由其批量传输至可观测性后端。访问日志根据流量量级进行自适应采样；指标始终输出但进行预聚合；链路追踪以低百分比采样，对错误进行基于尾部的采样。请求路径不会等待可观测性写入。

面试官： 多区域？

候选人： 每个区域运行独立的网关集群。Geo-DNS 或 anycast 将用户引导至最近的区域。路由和策略是全局状态，控制平面按区域进行复制。区域可通过撤回 anycast 广播或切换 DNS 进行故障转移。除非显式配置了故障转移，否则避免跨区域后端调用。

面试官： 网关中有什么，专用服务中有什么？

候选人： 网关中：TLS、认证验证、路由、限流执行（含缓存决策）、请求/响应转换。专用服务中：身份提供商、权威限流存储、服务发现、WAF 规则引擎、可观测性后端。原则：保持网关精简高效，将重型或专业逻辑推送至专用服务。

面试官： 总结。

候选人： 无状态数据平面加独立控制平面。版本化配置快照通过流式 RPC 分发；原子配置替换；分阶段发布以保障安全。认证、限流、转换作为过滤器，支持按路由配置。异步可观测性。按路由超时和熔断器控制后端故障影响。多区域部署为具备副本化控制平面的独立集群。最难保证正确性的问题是配置分发；最难优化的性能问题是将每个过滤器开销控制在亚毫秒以内。

3. 练习备注

五分钟版本

  1. 明确范围：TLS、认证、限流、路由、可观测性；按租户配置。
  2. 不变量：亚毫秒级开销，永不成为单点故障。
  3. 架构：无状态数据平面 + 控制平面。
  4. 过滤器链：TLS、认证、限流、路由、转换、可观测性。
  5. 配置：版本化快照、原子替换、分阶段发布。
  6. 将限流和身份认证委托给专用服务。
  7. 按路由配置超时、熔断器、舱壁隔离。
  8. 多区域：独立集群；地理路由。

一分钟结尾陈述

API 网关是典型的数据平面/控制平面分离架构。数据平面无状态，运行按路由的过滤器链，不阻塞于任何可能故障的组件。控制平面拥有真实状态，通过分阶段发布传播版本化快照。认证和限流以过滤器形式集成，并调用专用子系统。按路由超时和熔断器控制后端故障影响。最难的部分是对安全敏感变更的配置分发速度。

最佳面试用语

  • "数据平面通过永不同步调用任何可能故障的组件来保持高速；故障降级为缓存或本地状态。"
  • "配置分发使用带原子替换的版本化快照，使正在处理中的请求永远不会看到部分配置。"
  • "认证验证使用缓存；只有 introspection 才会在请求路径上访问认证服务。"
  • "限流功能已委托出去；网关调用专用服务，并在本地使用近似值作为兜底。"
  • "基于路由的超时和熔断器，防止单个慢速后端引发级联故障。"
