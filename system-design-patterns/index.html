System Design Patterns

2026/05/31

A taxonomy of the fundamental hardnesses that recur across system design interviews. Where the other sections group by component, this one groups by what is actually hard — the underlying constraint that breaks naive designs. Most “design X” prompts are really 2–3 of these patterns stacked together.
